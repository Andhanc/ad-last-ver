# ⚠️ ВАЖНО: Безопасность конфигурации

## Предупреждение о .env.local

Файл `.env.local` содержит **конфиденциальные данные**, включая:
- Пароли LDAP (`LDAP_BIND_PASSWORD`)
- Учетные данные для доступа к серверу

**⚠️ Этот файл добавлен в git репозиторий по запросу, но это НЕ рекомендуется для production!**

## Рекомендации по безопасности

### Для production окружения:

1. **НЕ коммитьте реальные пароли в git**
   - Используйте `.env.local` только для разработки
   - На production используйте переменные окружения или Docker secrets

2. **Используйте Docker secrets или переменные окружения:**
   ```yaml
   # docker-compose.yml
   environment:
     LDAP_BIND_PASSWORD: ${LDAP_BIND_PASSWORD}  # Из переменных окружения хоста
   ```

3. **Используйте отдельный файл для production:**
   - Создайте `.env.production` (добавьте в `.gitignore`)
   - Используйте его только на сервере

4. **Регулярно меняйте пароли:**
   - Если пароль был скомпрометирован, немедленно смените его
   - Обновите `.env.local` после смены пароля

## Что делать, если пароль был скомпрометирован

1. Немедленно смените пароль LDAP
2. Обновите `.env.local` с новым паролем
3. Перезапустите приложение
4. Проверьте логи на предмет несанкционированного доступа

## Альтернативные способы хранения секретов

### Вариант 1: Переменные окружения на сервере
```bash
export LDAP_BIND_PASSWORD="your_password"
docker-compose up -d
```

### Вариант 2: Docker secrets (для Docker Swarm)
```yaml
secrets:
  ldap_password:
    external: true
```

### Вариант 3: Отдельный файл (не в git)
```bash
# Создайте .env.production на сервере
cp .env.local .env.production
# Отредактируйте .env.production
# Добавьте .env.production в .gitignore
```
